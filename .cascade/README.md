# .cascade 폴더

**목적**: Cascade AI와의 소통 개선을 위한 상태 관리 폴더

---

## 📁 파일 구조

```
.cascade/
├── README.md                   # 이 파일
├── PROJECT_STATUS.md           # 전체 프로젝트 상태 (자동 생성)
├── SESSION_CHECKLIST.md        # 세션 시작 체크리스트 (사용자 업데이트)
└── WORK_LOG.md                 # 작업 로그 (자동 업데이트)
```

---

## 📝 파일 설명

### 1. PROJECT_STATUS.md
**목적**: 전체 프로젝트 상태 파악

**내용**:
- 완료된 기능
- 미완성 기능
- 현재 작업
- 문제점
- 다음 단계

**업데이트**: 주요 작업 완료 시 자동

**사용**:
- Cascade AI가 세션 시작 시 자동으로 읽음
- 프로젝트 전체 상태 파악
- 기존 구현 확인

---

### 2. SESSION_CHECKLIST.md
**목적**: 세션 시작 시 확인 사항

**내용**:
- 현재 작업
- 현재 문제점
- 다음 단계
- 중요 사항

**업데이트**: 사용자가 직접 업데이트

**사용**:
- Cascade AI가 세션 시작 시 자동으로 읽음
- 현재 작업 파악
- 문제점 파악
- 다음 단계 파악

---

### 3. WORK_LOG.md
**목적**: 일일 작업 내용 기록

**내용**:
- 날짜
- 작업 내용
- 소요 시간
- 결과

**업데이트**: 작업 완료 시 자동

**사용**:
- 작업 히스토리 추적
- 시간 관리
- 문제 패턴 파악

---

## 🎯 사용 방법

### Cascade AI (자동)
1. **세션 시작 시**:
   - `PROJECT_STATUS.md` 읽기
   - `SESSION_CHECKLIST.md` 읽기
   - 현재 상태 파악

2. **작업 중**:
   - 기존 코드 확인
   - 기존 패키지 활용
   - 변경 최소화

3. **작업 완료 시**:
   - `WORK_LOG.md` 업데이트
   - `PROJECT_STATUS.md` 업데이트
   - `SESSION_CHECKLIST.md` 업데이트 (사용자 확인 후)

---

### 사용자 (수동)
1. **작업 시작 시**:
   - `SESSION_CHECKLIST.md` 업데이트
   - 현재 작업 명시
   - 목표 명시

2. **작업 완료 시**:
   - `SESSION_CHECKLIST.md` 업데이트
   - 다음 작업 명시

3. **문제 발생 시**:
   - `SESSION_CHECKLIST.md`에 문제 기록
   - Cascade AI에게 알림
