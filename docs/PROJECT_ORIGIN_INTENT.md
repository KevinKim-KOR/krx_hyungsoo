## 3. 시스템 출력 정의 (PUSH로 받을 것)

시스템은 다음 3가지를 PC 분석 결과로 PUSH한다.

### 3.1. 보유 종목 추가매수/매도 의견
- 보유 종목에 대한 매매 의견 한 줄
- 무릎/머리/어깨 정량 기준 적용
- HOLD 상태인 종목은 PUSH 미발송
- 최종 매매 결정은 사용자가 한다

### 3.2. 신규 ETF 추천
- 잘 오르는 ETF 후보 알림
- 진입 시점 정량 분석은 시스템이 하지 않음
- 진입 여부 판단은 사용자가 AI 투자세션에서 토론 후 결정

### 3.3. 급락 ETF 경고
- 급락 상태 ETF 알림 ("건들지 마라" 신호)
- 단순 임계값 기반

## 시스템 책임 경계

- 시스템: 시장 흐름 판별, 정량 기준 적용, 알림 생성
- 사용자: 알림 해석, AI 투자세션 토론, 최종 매매 결정
- 매수/매도 API 자동화 없음
- "이상 감지" 자체는 사용자가 알림 보고 판단

## 4. 성공 기준 3개 (외부 검증형)

1. **친구의 긍정적인 놀람** — 친구에게 보여줄 때 "이거 대단하다"는 반응
2. **4070s의 동작** — ML 이외 작업을 못할 정도로 GPU가 돌아가는 상태
3. **와이프가 이해할 수 있는 UI** — 전문 용어 없이 읽히는 화면

**검증 우선순위 메모 (2026-05-11)**:
- 현재 검증 우선순위: 1차 검증자는 사용자 본인이다.
- 와이프 UI 이해도 검증은 사용자 본인이 운영 정착을 확인한 뒤 BACKLOG 에서 복귀한다 (docs/backlog/BACKLOG.md "와이프 UI 이해도 검증" 항목).
- 이 항목은 장기 성공 기준으로 유지한다 — 본 §4 의 3번 항목은 삭제하지 않는다.

## 6. 친구와 다르게 가고 싶은 부분

- **ML 엔진 검증 가능성**: 친구는 본인 판단 중심, 나는 AI+ML 교차 검증
- **합리적 판단 방식**: 여러 사람 조언 + AI 연구를 통한 판단
- **AI를 검증할 ML 보유**: AI에 맡기지 않고 검증 가능한 구조

단, ML 성능이 부족하면 결국 친구와 같은 길을 갈 가능성 인정.
