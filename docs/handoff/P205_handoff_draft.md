# P205 핸드오프 초안

## 1. P204 종료 요약

- P204는 `run_tune.py` 경계 안에서 튜닝 영속화, 구간 평가, 목적함수 교체를 마쳤다.
- 검산 파일(`trials_top20.csv`, `best_trial_segments.csv`, `tuning_summary.md`)이 추가됐다.
- Tune 카드 안에서 Top 5, 구간 평가, 승격 판정을 함께 보도록 정리됐다.
- 최신 승격 판정은 `REJECT` 이며, 현재 Full Backtest CAGR 기준이 미달 상태다.
- 다음 단계는 로직 재발명이 아니라 P205 범위 정의와 확장 우선순위 정리다.

## 2. P205 우선순위 후보

**우선순위 A**

- 검색 공간 확대 여부 검토
- quick / full 비교 UX 보강
- 승격 판정 기준 고도화

**우선순위 B**

- 유니버스 확장
- 체결 모델 현실화(익일 시가)
- 추천 / 알림 구조 검토

**우선순위 C**

- 뉴스 / 공포지수 같은 외생 변수 도입 검토

## 3. P205 시작 전 유지할 불변 조건

- `run_tune.py` 경계 유지
- Step 1~4B 산출물 보존
- 기존 `Run Tune` / `Run Backtest` 계약 보존
- 기존 승격 판정 / 검산팩이 깨지지 않게 진행

## 4. 첫 작업 권고

`P205 첫 작업은 유니버스 확대보다 검색 공간 확대와 quick/full 비교 UX 정리부터 시작한다.`
